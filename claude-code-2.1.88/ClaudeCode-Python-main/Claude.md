# CLAUDE.md

## 自动技能触发

以下规则用于自动判断何时应建议使用对应的 `.claude` 技能。**无需用户手动输入 `@`，直接执行或主动建议即可。**

### 代码库分析 → 使用 `@doc` 技能

当用户的请求涉及以下意图时，调用 `@doc` 并开始执行：

- 帮助我理解/熟悉这个代码库/项目
- 分析代码结构、架构、模块
- 梳理项目入口、依赖关系、数据流
- 代码审查前的概览
- 新建项目后的首次探索
- 任何表达为"看看这个项目"、"这是个什么项目"、"代码怎么组织的"等请求

### 代码修改/开发 → 使用 `@tdd` 技能

当用户的请求涉及以下意图时，调用 `@tdd` 并按 TDD 流程执行：

- 修改、开发、实现、添加功能
- 改写脚本、重构代码
- 修复 bug（需要先写复现测试）
- 新建模块/文件
- 任何涉及代码变更的操作

**例外**：如果用户明确说"快速实现不需要测试"，跳过 TDD 但需提示风险。

## 协作偏好

- 输出简洁，不重复用户已说过的内容
- 改动前先读现有代码，不盲目修改
- 问题优先用最小变更解决，不做过度设计
